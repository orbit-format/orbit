# Security Policy

## Supported Versions

We actively maintain the latest `main` branch and all published releases with a matching Git tag. Fixes are backported to the most recent minor release when feasible.

| Version | Supported          |
| ------- | ------------------ |
| main    | ✅                 |
| 0.x     | ✅ (latest patch)  |

Older releases will receive security fixes only at the maintainers' discretion.

## Reporting a Vulnerability

If you believe you have found a security vulnerability in Orbit, please create a [security advisory](https://github.com/orbit-format/orbit/security/advisories/new) on GitHub.

1. A description of the issue and why you believe it is a vulnerability.
2. Steps to reproduce or a proof of concept.
3. Any suggested remediation ideas, if you have them.

We will acknowledge receipt within 5 business days and provide status updates at least once every 10 business days until the issue is resolved. Please do not disclose the issue publicly until we have confirmed and released a fix.

## Preferred Languages

* English

## Public Disclosure

Once a fix is available, we will:

1. Coordinate an advisory and changelog entry.
2. Credit reporters who wish to be acknowledged.
3. Publish the fix via a tagged release.

Thank you for helping keep Orbit and its users safe.
